## 概念

`HTML5 Plus`（简称 **HTML5+** 或 **5+ App**）是一个**技术规范和一个运行环境**，而不是一个简单的 JavaScript 库。

我们可以把它理解为一个 **“增强版的浏览器内核”**，专门为移动 App 设计，让普通的 Web 技术（HTML、CSS、JavaScript）能够拥有媲美原生 App 的能力。

------

### 🎯 核心定义：它是什么？

1. 

   **一个技术规范 (Specification)**：

   - 

     它由 DCloud（数字天堂）公司制定，定义了一套丰富的 JavaScript API（就是我们常说的 `plus`API，如 `plus.runtime`, `plus.io`, `plus.camera`等）。

   - 

     这套规范详细描述了 Web 开发者如何通过 JS 代码去调用设备的原生功能，比如摄像头、文件系统、传感器、蓝牙等。

2. 

   **一个运行环境 (Runtime)**：

   - 

     为了让这套规范能在真实的手机上运行，需要一个底层的“引擎”来解析和执行这些 JS API，并与操作系统打交道。这个引擎就是 **HTML5+ Runtime**。

   - 

     你可以把它想象成一个“寄生”在原生 App 外壳里的、功能超级强大的浏览器。这个外壳负责提供运行环境和原生能力，内部的网页则用 `plus`API 来控制一切。

3. 

   **一种 App 开发模式 (Development Model)**：

   - 

     基于上述规范和环境，诞生了一种名为 **“5+ App”** 或 **“HTML5+ App”** 的开发模式。开发者使用 Web 技术编写应用，然后利用 HBuilder 等工具打包成真正的、可以上架商店的原生安装包（`.apk`或 `.ipa`）。

------

### 🧱 核心组成部分

HTML5+ 生态主要由三部分构成：

1. 

   **开发工具：HBuilder / HBuilderX**

   - 

     这是官方推荐的 IDE，它对 HTML5+ 开发进行了深度优化，提供代码提示、真机调试、云打包等功能。

2. 

   **运行环境：HTML5+ Runtime**

   - 

     这是打包进 App 的“内核”。当你用 HBuilder 云打包时，这个运行环境会被嵌入到你的 App 中。它负责将 `plus.xxx`的调用翻译成 Android/iOS 的原生代码并执行。

3. 

   **能力接口：HTML5+ API (即 `plus`API)**

   - 

     这是开发者直接接触的部分。一套统一的 JavaScript API，背后对应着海量的原生功能。这就是我们在 UniApp 的 App 端里能使用 `plus.io`、`plus.camera`的原因——因为 UniApp 的 App 端也集成了这个 HTML5+ Runtime。

------

### 🤔 HTML5+ 与 UniApp 是什么关系？

这是一个关键的区别点，很多人会混淆：

- 

  **HTML5+ (5+ App)**：是 **DCloud 提出的早期技术方案**。它非常强大，可以直接使用 `plus`API，灵活性极高，但**跨端能力弱**（主要针对 App 端，要做 H5 或小程序需要额外适配）。它的开发范式更偏向于“Web 工程师写 App”。

- 

  **UniApp**：是 **DCloud 推出的更现代、更强大的框架**。它的目标是 **“Write Once, Run Anywhere”**（一次编写，多端运行）。

  - 

    **UniApp 的 App 端，其底层运行环境就集成了 HTML5+ Runtime**。所以你在 UniApp 的 App 端里能看到并使用 `plus`API。

  - 

    **但是，UniApp 在 `plus`API 之上又封装了一层更简洁、跨平台的 `uni`API**（如 `uni.request`, `uni.navigateTo`）。官方推荐优先使用 `uni`API，以保证代码的多端兼容性。

  - 

    UniApp 不仅解决了 App 端的问题，还完美支持了 H5、微信/支付宝/百度等所有小程序，以及快应用。

**简单比喻**：

- 

  **HTML5+** 像是一台功能强大但需要手动换挡的 **手动挡跑车**，能发挥全部性能，但驾驭难度高。

- 

  **UniApp** 像是一台 **智能的自动挡汽车**，它同样拥有强大的引擎（集成了 HTML5+），但为你提供了一个更简单、更安全的驾驶界面（`uni`API），并且还能轻松变成船（H5）或飞机（小程序）去适应不同路况。

------

### ✅ 总结

| 特性         | HTML5+ (5+ App)                  | UniApp                                   |
| ------------ | -------------------------------- | ---------------------------------------- |
| **本质**     | 一套规范和运行环境               | 一个基于 Vue.js 的跨端框架               |
| **核心能力** | 提供 `plus`API 调用原生功能      | 提供 `uni`API 和 `plus`API，实现多端发布 |
| **跨端能力** | **弱**，主攻 App 端              | **极强**，一套代码多端运行               |
| **开发范式** | 更接近 Web 开发，灵活度高        | 基于 Vue.js，组件化、工程化              |
| **关系**     | **UniApp 的 App 端底层使用了它** | **上层框架**，对其进行了封装和扩展       |

**结论**：

**HTML5+ 是 UniApp 在 App 端的基石和能力来源之一**。没有 HTML5+，UniApp 的 App 端就无法拥有那些强大的原生能力。但 UniApp 通过提供更优秀的架构和跨端方案，成为了当前更主流、更推荐的选择。现在新项目通常直接使用 UniApp，而 `plus`API 则作为 UniApp 在特定场景下（调用 `uni`API 无法实现的功能）的补充。
