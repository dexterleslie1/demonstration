## gcc参数

### -I/usr/include/libpng16

头文件搜索路径（Include Path），告诉编译器在这个目录下查找`#include <xxx.h>`的头文件。如果 没有 -I/usr/include/libpng16，而你又写了`#include <png.h>`，就会出现：fatal error: png.h: No such file or directory

### -L/usr/lib

库文件搜索路径（Library Path），告诉链接器去这个目录找 .so或 .a库文件。如果 没有 -L/usr/lib，但用了 -lpng16，可能会报：undefined reference to `png_create_write_struct'

### -lpng16

链接具体的库（Link Library），实际链接的库文件是：libpng16.so   （动态库）或 libpng16.a    （静态库）。

## gcc为什么有时不用写-I和-L？

在很多系统（如 Ubuntu）中：

```sh
sudo apt install libpng-dev
```

后，编译器自动知道路径，可以直接写：

```sh
# g++ main.cpp -I/usr/include/libpng -L/usr/lib -lpng -o main
g++ main.cpp -lpng -o main
```

原因是：

- /usr/include是默认头文件路径
- /usr/lib是默认库路径
