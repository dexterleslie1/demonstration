## fmt库是什么呢？

>说明：因为这个库是使用C++语言开发，所以C语言如果要调用这个库需要包装一层麻烦。

`fmt`（通常指 **{fmt}**）是一个用于 C/C++ 的**现代格式化库**，主要用来替代或增强 `printf / std::ostream`，特点是：**快、类型安全、接口简洁**。

---

### 1. fmt 是什么？

- 一个开源 C++ 格式化库
- 官网/源码：`github.com/fmtlib/fmt`
- 被 C++20 标准采纳（`std::format` 就是基于它）

一句话总结：  
**fmt 用来“把各种数据格式化成字符串”，比 printf 更安全，比 iostream 更快。**

---

### 2. 为什么不用 printf / cout？

#### printf 的问题
```c
printf("%d %f\n", a, b);
```
- 参数类型不对会直接 UB（未定义行为）
- 不支持自定义类型
- 可读性一般

#### iostream 的问题
```cpp
std::cout << a << " " << b << std::endl;
```
- 语法冗长
- 性能较差
- 格式化能力弱

---

### 3. fmt 的基本用法

#### 示例
```cpp
#include <fmt/core.h>

int main() {
    int x = 42;
    double y = 3.14;

    fmt::print("x={}, y={:.2f}\n", x, y);
}
```

输出：
```
x=42, y=3.14
```

特点：
- `{}` 自动推导类型
- 不会因类型错误崩溃
- 格式语法类似 Python

---

### 4. 常见格式写法（简表）

| 写法     | 含义            |
| -------- | --------------- |
| `{}`     | 默认格式        |
| `{:d}`   | 十进制整数      |
| `{:x}`   | 十六进制        |
| `{:.2f}` | 保留 2 位小数   |
| `{:<10}` | 左对齐，宽度 10 |
| `{:>10}` | 右对齐          |

---

### 5. 相比 printf 的优势

- 编译期检查格式字符串
- 支持自定义类型的 `formatter`
- 性能通常优于 iostream
- API 简单统一

---

### 6. 和 std::format 的关系

- C++20 的 `std::format` ≈ fmt 的子集
- fmt 功能更多、更新更快
- 老编译器也可以用 fmt

```cpp
// C++20
#include <format>
std::format("x={}", x);
```

---

### 7. 适合用在什么场景？

- 日志系统
- 高性能文本输出
- 需要类型安全的格式化
- 替代 printf / sprintf

### 8. 示例

详细用法请参考本站示例：https://gitee.com/dexterleslie/demonstration/blob/main/demo-c++/demo-fmtlib.cpp

安装fmt库

```sh
sudo apt update
sudo apt install libfmt-dev

# 检查是否安装成功
pkg-config --modversion fmt
```

demo-fmtlib.cpp

```c++
#include <fmt/core.h>

int main() {
    int a = 42;
    double b = 3.14159;

    fmt::print("a = {}\n", a);
    fmt::print("b = {:.2f}\n", b);
    fmt::print("Hello, {}!\n", "world");

    return 0;
}
```

编译

```sh
g++ demo-fmtlib.cpp -o main -lfmt
```

运行

```sh
./main
```

## gio库是什么呢？

在 C 语言开发语境下，**Gio**（通常指 `libgio` 或 `gio-2.0`）是 **GLib** 库的一个核心子模块。

简单来说，它是 GNOME 项目提供的一套**现代化的、高层的输入/输出（I/O）和文件系统抽象库**。它的出现主要是为了替代老旧且复杂的 `GnomeVFS`，为 C 语言开发者提供更简洁、更强大的 API 来处理文件和流。

以下是关于 C 语言 Gio 库的详细介绍：

### 1. 核心定位：GLib 的一部分
Gio 并不是一个完全独立的库，它通常随 **GLib**（GNOME 的基础实用库）一起发布和安装。如果你在 Linux 系统上开发 C 程序，你很可能已经安装了它。
*   **头文件位置**：通常在 `/usr/include/gio-unix-2.0/` 或类似路径下。
*   **依赖关系**：它依赖于 `glib-2.0` 和 `gobject-2.0`。

### 2. 主要功能与特性
Gio 的设计目标是提供一个**虚拟文件系统（VFS）**，这意味着它把各种数据源（本地文件、网络文件、设备文件）都抽象成统一的接口。

*   **虚拟文件系统 (VFS)**：
    *   它允许你用同样的方式操作本地硬盘文件和远程网络文件（如通过 HTTP, FTP, SFTP 等协议）。
    *   核心对象是 `GFile`，它代表一个文件或目录的句柄，支持跨平台的路径处理。
*   **异步 I/O 支持**：
    *   这是 Gio 的一大亮点。它深度集成了 GLib 的**主循环（Main Loop）**，使得开发者可以轻松编写非阻塞的异步 I/O 代码，这对于保持 GUI 应用程序的响应速度至关重要。
*   **数据流抽象**：
    *   提供了 `GInputStream` 和 `GOutputStream` 接口，类似于 Java 或 C# 的流，支持过滤流（如压缩、缓冲）的链式操作。
*   **D-Bus 集成**：
    *   Gio 内置了对 **D-Bus** 的支持（`GDBus`），这是 Linux 桌面环境下进程间通信（IPC）的标准机制。你可以直接用 Gio 来发送和接收 D-Bus 消息，无需额外库。
*   **其他功能**：
    *   **网络监听**：`GSocket` 等类提供了跨平台的套接字编程接口。
    *   **应用信息**：`GDesktopAppInfo` 可以读取 `.desktop` 文件，用于获取 Linux 系统中应用程序的图标、名称和执行命令等信息。

### 3. 为什么使用它？
在 C 语言中处理文件和网络通常需要使用底层的 POSIX API（如 `open`, `read`, `write`, `socket`），这些 API 在不同操作系统上可能有差异，且难以处理异步操作。

**Gio 的优势在于：**
*   **跨平台一致性**：屏蔽了 Windows、Linux 和 macOS 在文件系统路径和权限上的差异。
*   **面向对象**：利用 GObject 系统，提供了面向对象的 C 语言编程风格，代码结构更清晰。
*   **功能丰富**：集成了文件、网络、D-Bus 和设置存储（`GSettings`）等多种功能。

### 4. 代码示例概念
如果你使用 Gio，代码风格通常是这样的（伪代码）：

```c
#include <gio/gio.h>

int main() {
    // 创建一个指向文件的对象
    GFile *file = g_file_new_for_path("/path/to/file.txt");
    
    // 读取文件内容（异步或同步）
    char *contents;
    g_file_load_contents(file, NULL, &contents, NULL, NULL, NULL);
    
    g_print("File content: %s\n", contents);
    
    // 记得释放内存（GObject 特性）
    g_free(contents);
    g_object_unref(file);
    
    return 0;
}
```

### 总结
**C 语言的 Gio 库**就是 **GLib 的 I/O 模块**。如果你正在开发 GNOME 应用程序，或者需要一个强大的库来处理**跨平台文件操作**、**异步流**或**D-Bus 通信**，Gio 是标准的解决方案。
