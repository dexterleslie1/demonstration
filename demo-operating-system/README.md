## UEFI、EFI、BIOS区别

**UEFI 是现代计算机的“启动固件”，而 BIOS 是它的老前辈。EFI 是 UEFI 的前身，现在基本可以看作是同义词。**

下面我们来详细解释它们的区别。

### 一句话总结

*   **BIOS**： 古老、过时的启动固件，界面简单，功能有限。
*   **UEFI**： BIOS 的现代继任者，更像一个轻量级的操作系统，功能强大，图形界面友好。
*   **EFI**： UEFI 的前身，由英特尔开发。现在通常用 UEFI 来统称。

---

### 详细对比表格

| 特性             | BIOS                                                         | UEFI                                                         |
| :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **全称**         | **B**asic **I**nput/**O**utput **S**ystem（基本输入输出系统） | **U**nified **E**xtensible **F**irmware **I**nterface（统一可扩展固件接口） |
| **诞生年代**     | 1975年（IBM PC）                                             | 2005年左右                                                   |
| **用户界面**     | 通常是蓝底白字的纯文本界面，只能用键盘操作。                 | 支持图形化界面（GUI），可使用鼠标，支持多语言。              |
| **启动模式**     | 仅支持 **Legacy** 模式。                                     | 支持 **UEFI** 模式和 **Legacy**（兼容）模式。                |
| **磁盘分区**     | 使用 **MBR** 分区方案。                                      | 默认使用 **GPT** 分区方案。                                  |
| **磁盘容量支持** | MBR 限制：最大支持 **2.2TB** 的启动硬盘。                    | GPT 无此限制，可支持巨大的硬盘容量（如 9.4 ZB）。            |
| **启动过程**     | 开机 → **POST**（加电自检） → 按顺序读取硬盘最开始的 **MBR** 扇区中的引导代码 → 启动系统。 | 开机 → 初始化硬件 → 直接读取 **EFI 系统分区（ESP）** 中的 **.efi** 引导文件 → 启动系统。更直接、快速。 |
| **安全性**       | 几乎没有内置的安全启动机制。                                 | 支持 **Secure Boot**（安全启动），可防止恶意软件在启动前加载，增强系统安全。 |
| **驱动支持**     | 驱动集成在固件中，更新困难。                                 | 驱动可以独立的 **.efi** 文件形式存放在硬盘上，更新灵活。     |
| **扩展性**       | 固件功能固定，难以扩展。                                     | “可扩展”，允许厂商添加特定功能的应用程序。                   |

### 核心区别详解

#### 1. 启动过程与磁盘分区（最关键的区别）

*   **BIOS + MBR**：
    *   BIOS 在启动时，会读取硬盘的第一个扇区（512字节），也就是**主引导记录（MBR）**。MBR 里包含了一小段引导代码和分区表信息。
    *   MBR 分区表的设计非常古老，它只能记录 **4个主分区**，并且最大只能支持 **2.2TB** 的硬盘。对于现代的大容量硬盘，这是一个致命的限制。
    *   启动过程是“链式”的，一环扣一环，容易出错。

*   **UEFI + GPT**：
    *   UEFI 不再依赖 MBR，而是会在硬盘上创建一个专门的 **EFI 系统分区（ESP）**，这是一个 FAT32 格式的小分区，里面存放着操作系统的引导程序（.efi 文件）。
    *   UEFI 使用 **GUID 分区表（GPT）**。GPT 没有分区数量和硬盘容量的限制（理论上），非常现代和健壮。
    *   启动过程更像是直接“调用”一个程序，更加简洁可靠。

#### 2. 用户界面与易用性

*   **BIOS**： 设置界面非常简陋，通常只能通过键盘的上下左右键操作，体验很差。
*   **UEFI**： 可以提供图形化的界面，支持鼠标操作，甚至可以集成一些简单的工具（如硬件诊断、浏览器等），用户体验好很多。

#### 3. 安全性

*   **BIOS**： 几乎没有安全措施，任何引导程序都可以在启动初期运行，容易感染引导型病毒或 rootkit。
*   **UEFI**： 支持 **安全启动（Secure Boot）** 功能。它要求所有在启动阶段加载的软件（如驱动、引导程序）都必须经过数字签名验证。这可以有效防止恶意软件篡改启动过程，是 Windows 8 及以上版本系统的重要安全特性。

### 如何判断你的电脑是 UEFI 还是 BIOS？

在 Windows 系统中：
1.  按 `Win + R`，输入 `msinfo32` 并回车，打开“系统信息”窗口。
2.  查看右侧的 **“BIOS 模式”** 一项。
    *   如果显示 **“传统”**，说明你正在使用 BIOS（Legacy）模式。
    *   如果显示 **“UEFI”**，说明你正在使用 UEFI 模式。

### 总结与现状

*   UEFI 是 BIOS 的彻底革新和替代品，它在性能、安全性、兼容性和功能性上都全面优于古老的 BIOS。
*   现在市面上所有的新电脑（过去10年内的产品）基本都使用 UEFI 固件，但为了兼容旧硬件（如旧硬盘、旧操作系统），UEFI 通常都提供一个 **Legacy 启动模式**（或叫 CSM 兼容性支持模块）来模拟 BIOS 环境。
*   在安装现代操作系统（如 Windows 10/11, 较新的 Linux 发行版）时，强烈建议使用 **UEFI + GPT** 的模式，以获得更快的启动速度、更好的安全性和对未来硬件的支持。
